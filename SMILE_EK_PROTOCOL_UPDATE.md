## 🔧 核心功能

### 1. 设备 ID 配置

- 默认设备 ID：**30**
- 可在配置页面修改设备 ID
- 系统自动替换指令中的 ID

**示例**:
```
配置 ID=30: #PWR30,1!  → 16 进制：23 50 57 52 33 30 2C 31 21
配置 ID=1:  #PWR1,1!   → 16 进制：23 50 57 52 31 2C 31 21
配置 ID=99: #PWR99,1!  → 16 进制：23 50 57 52 39 39 2C 31 21
```

### 2. ID 自动替换

系统会自动将指令中的设备 ID 替换为配置值:

```python
# 原始指令 (配置中)
payload_str: "#PWR0,1!"
payload_hex: "23 50 57 52 30 2C 31 21"

# 如果设备 ID 配置为 30
# 实际发送:
payload_str: "#PWR30,1!"
payload_hex: "23 50 57 52 33 30 2C 31 21"
```

### 3. 协议支持

- ✅ **smile_ek_tcp**: TCP 网口 / 串口服务器透传
- ✅ **smile_ek_com**: 直插本机串口 COM
- ✅ **pjlink**: PJLink 标准网络协议

## 💡 注意事项

### 指令格式

1. **控制指令必须以 `!` 结尾** (0x21)
   - 例如：`#PWR0,1!` = `23 50 57 52 30 2C 31 21`

2. **查询指令必须以 `?` 结尾** (0x3F)
   - 例如：`#PWR0,?` = `23 50 57 52 30 2C 3F`

3. **设备 ID 自动替换**
   - 配置中的指令使用 `0` 作为 ID 占位符
   - 实际发送时自动替换为配置的设备 ID

### 网络配置

- 串口服务器 IP: `192.168.50.72`
- 串口服务器端口：`502`
- 确保上位机能访问串口服务器

### 故障排查

1. **投影机无反应**
   - 检查设备 ID 是否正确
   - 确认网络连接正常
   - 查看日志窗口确认指令已发送

2. **指令格式错误**
   - 确认指令以 `!` 或 `?` 结尾
   - 检查 16 进制和字符串是否匹配
   - 运行验证脚本检查
