# 命令格式选择功能 - 使用说明

## 🎯 功能描述

现在点击"**📥 一键加载标准命令集**"按钮后，会弹出格式选择对话框，让您选择使用**字符串格式**还是**16 进制格式**来加载所有命令。

## 🚀 使用步骤

### 1️⃣ 点击"一键加载"

在投影机配置页面，点击蓝色按钮:
```
[📥 一键加载标准命令集]
```

### 2️⃣ 选择格式

弹出对话框，两个选项:

**选项 A: 字符串格式** (推荐)
- 按钮：`📝 字符串格式 (如：#1POWR 1)`
- 特点：
  - ✅ 可读性强，容易理解
  - ✅ 方便手动修改
  - ✅ 适合 ASCII 命令
  - 示例：`#1POWR 1`、`#1SOUR 17`

**选项 B: 16 进制格式**
- 按钮：`🔢 16 进制格式 (如：23 31 50 4F...)`
- 特点：
  - ✅ 原始 RS232 指令
  - ✅ 适合非 ASCII 命令
  - ✅ 精确控制每个字节
  - 示例：`23 31 50 4F 57 52 20 31 0D`

### 3️⃣ 加载完成

点击后自动加载所有命令，并显示成功提示:
```
✅ 成功加载 18 条字符串格式的命令！
```

或

```
✅ 成功加载 18 条 16 进制格式的命令！
```

## 📊 格式对比

### 字符串格式示例:

| 功能 | 指令 (字符串) |
|------|--------------|
| 开机 | `#1POWR 1` |
| 关机 | `#1POWR 0` |
| 切换至 PC | `#1SOUR 1` |
| 切换至 HDMI1 | `#1SOUR 17` |

**优点**:
- 直观，一眼就能看出是什么意思
- 容易记忆和修改
- 可以直接复制粘贴

**缺点**:
- 仅适用于 ASCII 字符集
- 某些特殊字符可能无法表示

---

### 16 进制格式示例:

| 功能 | 指令 (16 进制) |
|------|---------------|
| 开机 | `23 31 50 4F 57 52 20 31 0D` |
| 关机 | `23 31 50 4F 57 52 20 30 0D` |
| 切换至 PC | `23 31 53 4F 55 52 20 31 0D` |
| 切换至 HDMI1 | `23 31 53 4F 55 52 20 31 31 0D` |

**优点**:
- 精确控制每个字节
- 可以表示任意二进制数据
- 适合非 ASCII 命令

**缺点**:
- 不易读，需要转换才能理解
- 修改时需要计算字节
- 占用更多显示空间

## 💡 选择建议

### 推荐使用 **字符串格式** 的场景:

✅ 投影机支持 ASCII 命令 (如视美乐 EK 系列)  
✅ 需要经常手动修改命令  
✅ 团队协作，需要其他人也能看懂  
✅ 快速调试和测试  

### 推荐使用 **16 进制格式** 的场景:

✅ 需要精确控制每个字节  
✅ 命令包含非 ASCII 字符  
✅ 与硬件规格书保持一致  
✅ 特殊协议要求

## 🔄 切换格式

如果加载后发现格式不合适，可以:

### 方法 1: 重新加载
1. 删除所有命令 (点击每个命令的"🗑️ 删"按钮)
2. 重新点击"📥 一键加载标准命令集"
3. 选择另一种格式

### 方法 2: 手动修改
1. 找到需要修改的命令
2. 直接修改"16 进制/字符 指令"字段
3. 修改"指令格式"下拉框

## ⚙️ 技术细节

### 后台处理逻辑:

```javascript
// 用户选择格式后
function confirmLoadCommands(pIdx, brandId, format) {
    fetch(`/api/projector/brand_commands?brand_id=${brandId}`)
        .then(res => res.json())
        .then(data => {
            const cmds = data.commands || [];
            
            // 根据用户选择的格式加载命令
            conf.projectors[pIdx].commands = cmds
                .filter(c => c.visible !== false)
                .map(c => ({
                    id: c.id,
                    name: c.name,
                    // 关键：根据选择的格式获取对应的 payload
                    payload: format === 'hex' ? c.payload_hex : c.payload_str,
                    format: format,  // 记录格式
                    show_on_home: c.show_on_home || false,
                    sort: c.sort || 99,
                    icon: c.icon || '🔌'
                }));
            
            renderAll();
            alert(`✅ 成功加载 ${count} 条${format === 'hex' ? '16 进制' : '字符串'}格式的命令！`);
        });
}
```

### 数据流:

```
用户点击"一键加载"
    ↓
弹出格式选择对话框
    ↓
用户选择格式 (str/hex)
    ↓
调用 confirmLoadCommands(pIdx, brandId, format)
    ↓
请求 API: /api/projector/brand_commands?brand_id=xxx
    ↓
后台返回品牌命令库 (包含 payload_str 和 payload_hex)
    ↓
前端根据选择的格式提取对应的 payload
    ↓
渲染到页面
    ↓
显示成功提示
```

## ⚠️ 注意事项

1. **格式一旦选择，可以修改**
   - 选择后不是固定的
   - 可以手动修改单个命令的格式
   - 也可以重新加载全部命令

2. **两种格式效果相同**
   - 字符串 `#1POWR 1` 
   - 16 进制 `23 31 50 4F 57 52 20 31 0D`
   - 实际发送的字节完全一样

3. **建议统一格式**
   - 同一个投影机建议使用同一种格式
   - 便于管理和维护
   - 避免混淆

4. **某些品牌可能只有一种格式**
   - 如果品牌库中只提供了某种格式
   - 系统会自动使用可用的格式
   - 不影响正常使用
