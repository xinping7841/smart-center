## 1. 连接参数

### 1.1 RS232

官方文本已确认串口参数如下：

- 波特率：`19200`
- 数据位：`8`
- 停止位：`1`
- 校验位：`None`

直通线连接说明：

- `PIN3收 -> PIN3发`
- `PIN2发 -> PIN2收`
- `PIN5地 -> PIN5地`

### 1.2 TCP/IP

根据现场信息，网络方式连接参数为：

- IP：`192.168.50.53`
- 端口：`8080`

说明：

- 网络模式大概率为透明传输，直接发送与 RS232 相同的十六进制帧。
- 即：TCP 下仍然发送原始控制字节流，不需要额外包头。

## 2. 基本帧格式

## 2.1 常规控制帧

根据官方代码表与上位机生成结果，常规控制帧格式为：

```text
55 AA MM D1 D2 CS
```

字段说明：

- `55`：固定帧头
- `AA`：设备地址（ID）
- `MM`：模式/命令码
- `D1`：数据字节1
- `D2`：数据字节2
- `CS`：校验字节

## 2.2 校验算法

已确认校验方式为异或校验：

```text
CS = AA XOR MM XOR D1 XOR D2
```

注意：

- `帧头 0x55` 不参与异或计算。

示例：

```text
开启第1路（ID=0x01）
55 01 A0 00 01 A0

校验 = 01 XOR A0 XOR 00 XOR 01 = A0
```

## 2.3 扩展返回帧

根据状态查询与机器查询截图，设备还存在两种扩展返回帧：

- `57` 开头：状态查询返回
- `56` 开头：机器信息/时间参数类长帧返回

这部分已可用于解析，但个别长帧字段建议后续抓包再逐字节固化。

## 3. 地址规则

设备支持多地址，文本中明确给出了 `ID1 / ID2 / ID3` 的命令示例。

常见地址：

- `0x01`
- `0x02`
- `0x03`

广播/全体机器相关命令中，截图与文本中出现过地址 `0x00`。

例如：

```text
所有机器同时开启：55 00 AC 00 00 AC
所有机器同时关闭：55 00 AD 00 00 AD
读取机器地址：    55 00 AB 00 00 AB
```

## 5. 模式行为说明

根据官方《模式更改说明》，设备支持 3 种客户模式。

## 5.1 时序模式

时序模式下，以下功能有效：

1. 所有机器开启 / 所有机器关闭
2. 查询是否锁定
3. 某一路开启 / 关闭
4. 级联开 / 关
5. 级联 / 非级联保存
6. 定时开 / 关
7. 某一机器所有开启 / 关闭
8. 某一机器顺序开启 / 逆序关闭
9. 锁定保存 / 不锁定保存
10. 时间间隔设置

## 5.2 互锁模式

互锁模式下，共四组：

- `1-2` 互锁
- `3-4` 互锁
- `5-6` 互锁
- `7-8` 互锁

## 5.3 继电器模式

继电器模式下：

- 仅能通过机器本体轻触操作各路通道。
- 某些时序相关协议命令不再可用。

## 5.4 模式切换注意事项

官方特别说明：

1. 在做模式转换前，应先关闭所有通道。
2. 机器在时序模式启动后，可以且只能通过指令进行模式转换。
3. 在非时序模式下，与时序相关的某些指令不可控。

## 6. 状态查询返回解析

查询命令：

```text
55 ID A8 00 00 CS
```

根据截图，返回帧格式为：

```text
57 ID B0 D1 D2 D3 CS
```

其中：

- `57`：状态返回帧头
- `ID`：设备地址
- `B0`：状态返回命令码
- `D1`：系统状态字节
- `D2`：8路输出状态位图
- `D3`：8路屏蔽/保留状态位图

## 6.1 D1 解析

根据截图说明，`D1` 至少包含以下位：

- `bit0`：LCD 状态
  - `0 = 开`
  - `1 = 关`
- `bit1`：锁定状态
  - `0 = 未锁定`
  - `1 = 锁定`
- `bit2`：当前运行状态标志
  - 用于表示启动/关闭过程状态

说明：

- `bit2` 的具体含义建议后续抓一次完整状态返回包再固化。

## 6.2 D2 解析

`D2` 为 8 路输出状态位图，位定义如下：

- `b7..b0` 对应 `CH1..CH8`
- `1 = 关闭`
- `0 = 开启`

示例：

- `D2 = 0x7F`：表示第1路开启，其它关闭
- `D2 = 0x5B`：根据截图备注，可解析为部分通道开启、部分关闭

建议实际程序解析方式：

```text
CHn = ((D2 >> (8 - n)) & 0x01) == 0  => 开启
```

更直观写法：

- `bit7 -> CH1`
- `bit6 -> CH2`
- `bit5 -> CH3`
- `bit4 -> CH4`
- `bit3 -> CH5`
- `bit2 -> CH6`
- `bit1 -> CH7`
- `bit0 -> CH8`

## 6.3 D3 解析

`D3` 为 8 路屏蔽/状态扩展位图。

目前从截图能确认：

- `b7..b0` 对应 `CH1..CH8`
- 与屏蔽/非屏蔽有关
- `0xAA / 0x55` 也出现在相关屏蔽控制命令中

但 `D3` 每一位的最终定义还建议后续抓包确认。

## 7. 机器查询返回解析

查询命令：

```text
55 ID A9 00 00 CS
```

根据截图，若机器存在，返回帧以 `56` 开头，命令码为 `A7`。

大致结构为：

```text
56 ID A7 ... CS
```

返回内容包含：

- 开机延时：LCD + CH1~CH8
- 关机延时：LCD + CH1~CH8
- 级联参数
- 锁定状态

## 7.1 时间字段编码规则

根据截图说明，时间字节编码规则如下：

- 最高位为 `1`：单位为“分钟”
- 最高位为 `0`：单位为“0.5秒”
- 低 7 位为数值

例如：

- `0x05`：`5 x 0.5秒 = 2.5秒`
- `0x85`：`5分钟`

## 8. 时间间隔设置

根据“间隔时间设置”截图：

- 使用模式码 `A7`
- 为长帧
- 帧头表现为 `56`
- 用于连续发送 9 路开启延时与 9 路关闭延时

这里的 9 路通常可理解为：

- `LCD`
- `CH1`
- `CH2`
- `CH3`
- `CH4`
- `CH5`
- `CH6`
- `CH7`
- `CH8`

说明：

- 这部分字段位序在未抓包前，建议按官方上位机实测包再最终落代码。

## 10. 推荐接入方式

如果要接入现有中控/运维平台，建议按下面顺序实现。

## 10.1 控制功能优先级

第一阶段建议先实现：

1. 单路开启
2. 单路关闭
3. 顺序全部开启
4. 顺序全部关闭
5. 状态查询

这样已经可覆盖大部分运维控制场景。

## 10.2 状态轮询建议

建议主轮询命令使用：

```text
55 ID A8 00 00 CS
```

原因：

- 返回短
- 可以获得 8 路当前开关状态
- 还可解析锁定和运行状态

## 10.3 初次识别建议

新设备接入时，可按下列顺序：

1. 读取机器地址：`55 00 AB 00 00 AB`
2. 读取机器信息：`55 ID A9 00 00 CS`
3. 周期性轮询状态：`55 ID A8 00 00 CS`

## 10.4 TCP 与串口统一驱动

由于 TCP 侧大概率是透明串口服务器，建议程序层统一采用：

- 一套协议编码器
- 两种传输通道
  - `TCP socket`
  - `Serial`

这样后续切换 `RS232 / 串口服务器 / 设备直连网络` 时，协议层无需改动。
